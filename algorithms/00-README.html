The Functional Parsing Library

Parsing Algorithms (ch. 5, 6 and 7)

To use a parsing algorithm described in these chapters, you first of all need the following files:
  • OrdSet.hs   (from the parent directory)
  • OrdMap.hs   (from the parent directory)
  • Grammar.hs
The modules that are called ...ToChart.hs  also make use of the FinalChart.hs module.

Chart parsing (ch. 5)

There are three different implementations of chart parsing:
  1. ChartParserOriginal.hs   (the unoptimized code in sections 5.3-5.5)

  2. ChartParserOptimized.hs   (with the optimizations suggested in section 5.6)

  3. ChartParserToChart.hs   (returning a chart of passive edges; optimized code)

Generalized LR parsing (ch. 6)

To use the different LR implementations, you first need the following three modules:
  • LRTableOriginal.hs
  • LRTableOptimized.hs
  • LRAugment.hs
Then you can choose any of four different implementations:
  1. LRParserDF.hs   (depth-first parsing from sections 6.2.2 and 6.3.2)

  2. LRParserBF.hs   (breadth-first parsing from sections 6.2.3 and 6.3.2)

  3. LRParserT.hs   (approximative Tomita parsing from sections 6.2.4-5 and 6.3.3-4)

  4. LRParserToChart.hs   (approximative Tomita parsing to a chart of passive edges)

CYK parsing (ch. 7)

There are three different implementations of CYK parsing:
  1. CYKParserOriginal.hs   (the unoptimized code in section 7.3)

  2. CYKParserOptimized.hs   (with optimized lookup of grammar productions)

  3. CYKParserUnitToChart.hs   (returning a chart of passive edges; optimized code; also handles unit productions)
