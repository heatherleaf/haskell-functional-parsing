The Functional Parsing Library

Parsers (ch. 3 and 4)

To use a parser described in these chapters, you first of all need the following files:
  • OrdMap.hs   (from the parent directory)
  • Parser.hs
Then you can create the following parsers by adding the given files:
  • Standard s
    (the standard parser, sec. 3.2): Standard.hs
  • ContTrans (Standard s)
    (the standard continuation parser, sec. 3.3.2): Standard.hs, ContTrans.hs
  • ContTrans (StandardEndo s)
    (the endomorphism continuation parser, sec. 3.3.4): StandardEndo.hs, ContTrans.hs
  • StackTrans (Standard s)
    (the standard stack continuation parser, sec. 3.4.1): Standard.hs, StackTrans.hs
  • StackTrans (StandardEndo s)
    (the endomorphism stack continuation parser, sec. 3.4.1): StandardEndo.hs, StackTrans.hs
  • Stream s
    (the stream processor parser, sec. 3.5.2): Stream.hs
  • ContTrans (Stream s)
    (the stream continuation parser, sec. 3.5.3): Stream.hs, ContTrans.hs
  • StackTrans (Stream s)
    (the stream stack continuation parser, sec. 3.5.4): Stream.hs, StackTrans.hs
  • Trie s
    (the trie parser, sec. 4.2.1): Trie.hs
  • AmbTrie s
    (the ambiguous trie parser, sec. 4.2.2): AmbTrie.hs
  • ExTrie s
    (the extended trie parser, sec. 4.3.3): ExTrie.hs
  • AmbExTrie s
    (the ambiguous extended trie parser, sec. 4.3.4): AmbExTrie.hs
  • PairTrie (Standard s), PairTrie (ContTrans (Standard s)), PairTrie (ContTrans (StandardEndo s)), PairTrie (StackTrans (Standard s)), PairTrie (StackTrans (StandardEndo s))
    (the paired trie parser, sec. 4.4): PairTrie.hs, together with a subset of (Standard.hs, StandardEndo.hs, ContTrans.hs, StackTrans.hs)
The file ExampleParsers.hs exports all the parsers given above as one module.
